inicialmente, rodei o nmap com os parametros -sC -sV -v -T4 e foram encontradas diversas portas abertas e na interação com cada uma delas mostrava uma mensagem diferente relacionada a possuir ou não algo útil até que em uma delas havia uma referência sobre onde encontrar a key.

Ao abrir a página indicada para achar a key um arquivo é baixado automaticamente. Na tentatica de descobrir do que se trata o arquivo usei o comando file <nomedoarquivo> e se tratava de um arquivo que poderia ser analisado o comando strings para descobrir seu conteúdo e ali achamos a key.

Rodando o feroxbuster achamos a página home.php. Nela achei um local que deixava inserir comandos como se fosse um terminal linux. Ao digitar o comando ls apareceram diversos arquivos com a extensão .php e tive a ideia de testar se conseguiria executar um comando de reverse shell php. Assim, coloquei o netcat para escutar na porta 1234 e executei o comando php tendo nele o ip da minha máquina na vpn e a porta que estava usando para escutar. Dessa maneira, consegui spawnar uma reverse shell.

O terminal da rev shell abre no diretório /var/www/html que é o diretorio onde ficam os arquivos servidos pelo web server. Com o comando ls vemos diversos arquivos .php, dentre eles o validate.php e ao dar um cat nele vemos um script php de validação que deixa explicito o usuário e senha aceitos, sendo o usuário charlie.

No diretório /home/charlie encontramos uma chave privada em um dos arquivos que pode ser usada para login ssh com chave usando o usuário charlie. Para isso, copiamos o conteúdo do arquivo para um arquivo na máquina local e mudamos a permissão do arquivo com o comando chmod 600 para que possarmos usar como identidade do ssh.

Com o acesso ssh dando certo temos acesso aos arquivos que o usuario charlie tem permissão de acessar, incluindo a flag de user. Com o comando sudo -l descobrimos que o sudo está mal configurado e permite rodar vi como qualquer usuário e a partir dele podemos obter acesso root. Usando sudo vi, temos acesso a um editor vi e podemos executar ":!bash" que retorna para o terminal como usuário root. O vi pode executar comandos do sistema e todos eles rodam como root.

Indo até o diretório /root e dando o comando ls encontramos um código root.py. Ao ver como ele funciona, descobrimos que ele descriptografa uma mensagem usando a key que foi encontrada no início do desafio e revela a flag root.
